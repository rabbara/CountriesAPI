Tuto 1: Comment configurer un environement virtuel Python sur Windows 11?

Pré-requis: avoir installer une version de python supérieur ou égal à la version 3.8 et git sur sa machine

Introduction

Le but de ce tutoriel est de voir comment configurer notre envionnement virtuel pour notre projet CountriesAPI.

L'intérêt de configurer un environement propre à notre projet au lieu d'utiliser l'environnement global de python est simple. En cloisonnant, notre projet dans un environnement virtuel on peut lui définir les librairies et la version de python à à utiliser pour notre projet sans risquer d'avoir un conflit avec d'autres projets qui ont peut-être d'autres dépendances ou bien même une autre version de python.

Pour éviter les conflits avec d'autres projets, il faut qu'un script ou un projet soit dans un environnement virtuel qui lui est propre.

Initialisation du projet

Pour ce tutoriel on va créer un répertoire que l'on va nommer Projects où l'on va placer notre projet CountriesAPI.

Dans un terminal git Bash (MAJ+CTRL+click droit -> Git Bash)

  • Créer un dossier que l'on nomme Projects: mkdir Projects

  • Placez-vous dans le dossier Projects/: cd Projects

  • Récupérer le projet sur GitHub: git clone https://github.com/rabbara/CountriesAPI.git

Vous devriez avoir cette arborescence:

Projects/

CountriesAPI

Création de l'environnement virtuel

Pour créer un environnement virtuel python sur Windows, on execute la commande suivante: python -m venv <name>, où <name> correspond au nom du repertoire où l'on veut configurer notre environnement.

Par exemple, la commande python -m venv env va créer un environnement virtuel qui se nomme env.

Remarque: si vous souhaitez spécifier une versionde python particulière executer la commande python3.X -m venv <name> où X est la version que vous souhaitez.

Sur le terminal exécuter ces commandes:

  • Placez-vous dans le dosser CountriesAPI/: cd CountriesAPI

  • Créer un environnement qui se nomme venv: python -m venv venv

Vous devriez avoir un nouveau répertoire venv avec l'environnement virtuel pour votre projet.

Vous devriez avoir cette arborescence:

Projects/

CountriesAPI

venv

Activer l'environnement virtuel

Pour activer l'envrionnement virtuel que vous venez de créer, vous devez exéctuer sur le terminal la commande:

  • Sur un terminal GitBash: source venv/Scripts/Activate

  • Sur un terminal PowerShell: venv/Scripts/Activate

Vous devriez voir apparaitre un (venv), comme ci-dessous:

Vous êtes désormais dans l'environnement virtuel et toutes les commandes executées seront dans l'environnement virtuel. C'est-à-dire que les libraires installées se feront uniquement dans cette environnement.

Installer les librairies

Désormais, nous avons notre environnement en place. Nous pouvons installer les librairies nécessaires à notre projet.

Pour ce faire taper la commande suivante sur le terminal GitBash: pip install -r requirements.txt

Cette commande permet d'installer toutes les librairies listées dans le fichier requirements.txt. Vous pouvez mettre à jour ce fichier en listant les librairies installées avec la commande pip freeze > requirements.txt

Executer un script depuis votre environnement virtuel

Pour executer un script depuis l'environnement virtuel

  • Sur un terminal GitBash: <path_to_project>/venv/Scripts/python.exe <path_to_script>
